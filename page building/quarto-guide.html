Integrating R Quarto into Your Academic Website

This guide will help you create and integrate interactive R Quarto documents into your academic website.

1Setting Up Your Environment

First, ensure you have the necessary tools installed:

  • R: The programming language needed for analysis
  • RStudio: The IDE that provides a convenient interface for R
  • Quarto: The publishing system for scientific and technical content

Installation links:

  • Download R
  • Download RStudio
  • Download Quarto

2Creating Your Quarto Document

In RStudio, create a new Quarto document:

  1. Go to File > New File > Quarto Document
  2. Select HTML as the output format
  3. Save the file as blog.qmd in your project directory

Your Quarto document should start with YAML front matter that defines basic properties:

--- title: "blog Projects" author: "Heeyoung Lee" format: html: theme: cosmo css: custom.css toc: true toc-depth: 3 code-fold: true code-tools: true self-contained: true ---
Pro Tip: The self-contained: true option creates a single HTML file with all dependencies embedded, making it easier to integrate into your website.

3Creating Custom CSS for Styling

Create a file named custom.css in the same directory as your Quarto document to customize the appearance:

/* Add Bootstrap icons */ @import url("https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.1/font/bootstrap-icons.css"); /* Add Animate.css for animations */ @import url("https://cdnjs.cloudflare.com/ajax/libs/animate.css/4.1.1/animate.min.css"); /* General styling */ body { font-family: system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial; line-height: 1.6; } h1, h2, h3, h4, h5, h6 { color: #2d3748; } /* Make findings stand out */ .key-finding { background-color: #f8f9fa; padding: 1.25rem; border-radius: 8px; margin: 1.5rem 0; border-left: 4px solid #2d3748; } .finding-number { font-size: 2.5rem; font-weight: bold; color: #2d3748; line-height: 1; } /* Style for interactive elements */ .plotly, .leaflet { width: 100%; height: auto; margin: 1.5rem 0; }

This CSS will help your Quarto document match the style of your existing website.

4Adding Interactive Content

Quarto allows you to create interactive visualizations using R packages like plotly, leaflet, and DT. Here's an example of how to add an interactive plot:

```{r interactive-plot, fig.width=10, fig.height=6} # Load required libraries library(tidyverse) library(plotly) # Create sample data years <- 2010:2020 set.seed(123) depression_data <- data.frame( Year = rep(years, 4), Group = rep(c("Group A", "Group B", "Group C", "Group D"), each = length(years)), Rate = c( # Group A data 8.2 + cumsum(rnorm(length(years), 0.5, 0.2)), # Group B data 7.8 + cumsum(rnorm(length(years), 0.2, 0.1)), # Group C data 9.5 + cumsum(rnorm(length(years), 0.7, 0.2)), # Group D data 7.5 + cumsum(rnorm(length(years), 0.3, 0.1)) ) ) # Create the interactive plot p <- depression_data %>% ggplot(aes(x = Year, y = Rate, color = Group, group = Group)) + geom_line(size = 1.2) + geom_point(size = 3) + theme_minimal() + labs(title = "Depression Rates by Group") # Convert to plotly for interactivity ggplotly(p) ```

For interactive maps, you can use the leaflet package:

```{r interactive-map, fig.width=10, fig.height=6} library(leaflet) # Create a simple interactive map map <- leaflet() %>% setView(lng = -98.5795, lat = 39.8283, zoom = 4) %>% addTiles() %>% addMarkers(lng = -73.9665, lat = 42.6526, popup = "University at Albany, SUNY") map ```

5Rendering Your Quarto Document

Once you have created your content, you can render the Quarto document to HTML:

  1. In RStudio, click the "Render" button in the editor toolbar
  2. Alternatively, use the terminal or RStudio console to run: quarto render blog.qmd

This will create an HTML file (blog.html) that contains your interactive content.

Important: Make sure all required R packages are installed before rendering. You can install packages using install.packages("package_name").

6Integrating into Your Website

Now you can integrate the rendered HTML file into your GitHub Pages website:

  1. Create a new folder in your GitHub repository for your blog content (e.g., blog)
  2. Upload the rendered blog.html file to this folder
  3. If your Quarto document has external dependencies (images, data files), upload those as well

Your repository structure might look like this:

  • Your GitHub Repository
    • index.html
    • page building
      • about.html
      • contact.html
      • publications.html
      • teaching.html
      • cv.html
    • blog
      • blog.html
      • data.csv
      • images
    • images
    • pdf

7Adding Navigation to Your blog Page

Update your website's navigation to include a link to your new blog page:

  1. Open your index.html file
  2. Add a new navigation item in the navbar section:
<li class="nav-item"> <a class="nav-link" href="blog/blog.html"> <i class="bi bi-graph-up me-1"></i> Interactive blog </a> </li>

8Automating Updates with GitHub Actions (Optional)

If you want to automate the process of updating your blog page, you can use GitHub Actions to render your Quarto document whenever you push changes to your repository:

  1. Create a file named .github/workflows/render-quarto.yml in your repository
  2. Add the following content to the file:
name: Render Quarto on: push: paths: - 'blog/blog.qmd' - 'blog/custom.css' jobs: render: runs-on: ubuntu-latest steps: - name: Check out repository uses: actions/checkout@v3 - name: Set up R uses: r-lib/actions/setup-r@v2 - name: Install dependencies run: | R -e 'install.packages(c("tidyverse", "plotly", "leaflet", "htmlwidgets", "kableExtra", "DT"))' - name: Set up Quarto uses: quarto-dev/quarto-actions/setup@v2 - name: Render Quarto document run: | quarto render blog/blog.qmd - name: Commit results run: | git config --local user.email "actions@github.com" git config --local user.name "GitHub Actions" git add blog/blog.html git commit -m "Re-render Quarto documents" || echo "No changes to commit" git push origin || echo "No changes to push"

With this workflow, your Quarto document will be automatically rendered and the updated HTML file will be committed to your repository whenever you make changes to the QMD file.
